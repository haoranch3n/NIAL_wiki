Overview

The Cellpose 3D Pipeline performs automated multi-scale cell segmentation and 3D reconstruction on fluorescence microscopy images directly from OMERO. It uses the Cellpose deep learning model to detect cells at multiple sizes, then reconstructs them into 3D objects using 3DCellComposer — a versatile method for building consistent 3D cell segmentations from 2D slice-by-slice results. Finally, it quantifies per-cell fluorescence intensity for each channel.

At a Glance

Supported input formats: ND2, TIF, OME-TIFF
Channels used: DAPI (nuclear) + a secondary cytoplasmic channel (TH by default)
Output: A CSV file with per-cell statistics (attached to each source image in OMERO), plus an analysis visualization image imported into a new OMERO dataset.

What the Pipeline Does

The pipeline runs four major stages automatically. You do not need to run them individually — the system handles everything after you submit the job.

StageNameDescription
1PreprocessingExtracts individual 2D planes from the multi-dimensional image (one per Z-slice and channel). If a .roi file (ImageJ format) with the same name as the image exists, pixels outside the ROI polygon are automatically zeroed before extraction.
22D Cell SegmentationRuns Cellpose at multiple cell diameters to detect cells of varying sizes on each 2D plane (GPU-accelerated). Multi-scale masks are then combined, small objects filtered, and touching cells split.
33D Cell ReconstructionStacks 2D masks into 3D volumes and matches cell labels across Z-slices using 3DCellComposer to create consistent 3D cell IDs. Cells appearing in only one Z-plane are removed.
4QuantificationComputes mean fluorescence intensity per channel for every segmented 3D cell and writes a CSV file with per-cell statistics.

How to Run the Pipeline

Step 1: Place Your Data

Your images must be accessible on the shared filesystem at the following path:

/media/core/core_operations/OmeroData/{lab}/{user_name}/{project_name}/{dataset_name}/

Place your microscopy files (ND2, TIF, or OME-TIFF) into the appropriate dataset folder. If you are using ROI cropping, place the corresponding .roi files in the same folder (see ROI Cropping section below).

Step 2: Open OMERO.web and Select Images

Navigate to https://omeroplus.stjude.org and log in with your organizational credentials. Then select the data to process:

  • Individual images — select one or more images in the left panel
  • Entire datasets — select one or more datasets (all images inside will be processed)

Step 3: Launch the Script

  1. Click the Scripts icon (gear/wrench) in the top toolbar.
  2. Navigate to Processing & Analysis → Run Cellpose 3D.
  3. Configure the parameters (see table below).
  4. Click Run Script.

Step 4: Monitor Progress

After submission, click the thumbnail icon in the top-right area of OMERO.web and select Job Monitor to track your pipeline. In the Job Monitor, set the job type filter to HPC Jobs. The tracker shows progress for each phase:

PhaseComputeTypical Duration
SegmentationGPU~1–2 hours
Post-processingCPU~30 minutes
Attach ResultsCPU< 5 minutes
OMERO ImportCPU< 5 minutes

Actual runtimes depend on image size, number of diameters, and SLURM cluster queue availability.

Step 5: View Results

When the pipeline completes:

  • CSV file — attached to the original source image. Select the image, then on the right side of the screen expand the Attachments tab. The _cell_intensity_stats.csv file will be listed there and can be downloaded.
  • Analysis image — imported into a new dataset named {Dataset}_cellpose3D_{timestamp} under the same project. This OME-TIFF shows the raw channels overlaid with segmentation labels. Double-click the image to open it in the OMERO viewer for interactive inspection.

Parameters

ParameterDefaultDescription
Data_TypeImageProcess individual Images or all images in Datasets
IDs(required)Select the images or datasets to process
Diameter_Start50Minimum expected cell diameter in pixels
Diameter_End100Maximum expected cell diameter in pixels
Diameter_Step5Increment between diameters (smaller = more scales, longer runtime)
Modelcustomcustom = trained specifically for DAPI/TH mouse brain tissue; cyto2 = general-purpose Cellpose model
Min_Cell_Diameter40Cells smaller than this diameter (in pixels) are filtered out
Split_Threshold0.8Controls how aggressively touching cells are separated (0.0–1.0; higher = less splitting)
JI_Threshold0.3Jaccard Index threshold for matching cells across Z-slices (higher = stricter matching)
Filter_Single_SliceTrueRemove cells that appear in only one Z-plane (recommended for 3D analysis)

Choosing Diameter Range

The diameter range should bracket the expected cell sizes in your images:

  • Too narrow: May miss cells outside the range
  • Too wide: Increases runtime without improving results
  • Typical starting point: 50–100 with step 5 (11 diameters)
  • For smaller cells (e.g., 20x magnification): try 30–70
  • For larger cells (e.g., 10x magnification): try 70–150

ROI Cropping (Optional)

If you only want to analyze a specific region of your image, you can draw an ROI in ImageJ/FIJI and save it as a .roi file:

  1. Open your image in FIJI/ImageJ.
  2. Draw a freehand or polygon selection around the region of interest.
  3. Go to File → Save As → Selection... and save it with the same name as the image file but with a .roi extension.
    • Example: if your image is sample.nd2, save the ROI as sample.roi in the same folder.
  4. The pipeline will automatically detect the .roi file and zero out all pixels outside the ROI before processing.
Note: If no .roi file is found, the pipeline processes the full image. No action is needed — ROI cropping is entirely optional.

The CSV file ({image_name}_cell_intensity_stats.csv) contains one row per segmented cell with the following columns:

Note: Channel columns depend on the channels present in your image.

Analysis Image

An OME-TIFF image ({image_name}_analysis.ome.tif) is imported into OMERO showing the raw fluorescence channels combined with the segmentation labels and per-cell mean intensity overlays. This image can be viewed in OMERO.web or downloaded for further inspection.

Troubleshooting

Pipeline shows "Failed" in OMERO Jobs panel

  • Check the SLURM logs on the shared filesystem. Logs are in the SCRATCH directory under cellpose3d_{timestamp}/logs_*/.
  • Common causes: input image missing DAPI channel, file permissions, disk space.

No CSV attached to my image

  • Verify the pipeline completed all phases (check the Jobs panel).
  • The CSV is only attached after the final "Attach Results" phase succeeds.
  • If the image was in a group you don't own, the pipeline account may need write permissions.

Segmentation quality is poor

  • Too few cells detected: Try a wider diameter range or lower Min_Cell_Diameter.
  • Too many false detections: Increase Min_Cell_Diameter or use a narrower diameter range.
  • Cells not separated: Lower the Split_Threshold (e.g., 0.6).
  • Inconsistent 3D labels: Lower the JI_Threshold (e.g., 0.2) to allow more matches across Z-slices.

Pipeline is very slow

  • Reduce the diameter range (fewer diameters = faster).
  • Increase Diameter_Step (e.g., 10 instead of 5).
  • Processing time scales linearly with the number of diameters and the number of Z-slices.

FAQ

Q: What channels does the pipeline use?
A: The pipeline uses DAPI as the nuclear channel and TH as the secondary cytoplasmic channel. These are currently configured by the system administrator. Contact the Image Analysis Core if you need to use a different channel (e.g., AT8).

Q: Can I process ND2 files directly?
A: Yes. The pipeline accepts ND2, TIF, and OME-TIFF files. No conversion is needed before submitting.

Q: What happens if my image has no matching ROI file?
A: The pipeline processes the full image. ROI cropping is entirely optional.

Q: Can I re-run the pipeline on the same images?
A: Yes. Each run creates a new output dataset with a unique timestamp. Previous results are not overwritten.

Q: Who do I contact for help?
A: Contact the Image Analysis Core team (Haoran Chen, Jorge Gutierrez).
