1. What Is Cursor?

Cursor is an AI-assisted code editor based on VS Code. It helps you write, edit, and understand code using natural language. You can:

  • Ask it to make code changes across one or many files
  • Explain files, functions, or how a project works
  • Generate scripts or starter code
  • Get help with command-line tasks using the built-in terminal agent

2. Setup & Access

Cursor is available as a desktop app (full GUI) and a CLI agent (terminal only). Which one you can use depends on your environment:

2.1 Cursor Desktop App

Install Cursor on your local machine, then connect to remote servers via the built-in SSH remote feature.

Installation by platform macOS
  1. Download the .dmg from cursor.com.
  2. Drag Cursor.app into /Applications.
  3. Launch and sign in (see Login section below).
 Windows
  1. Download the .exe installer from cursor.com.
  2. Run the installer and follow the prompts.
  3. Launch and sign in.
 Linux
  1. Download the .AppImage (x86 version) from cursor.com.
  2. Right-click the downloaded file → Properties → Permissions tab → set access to Read and Write and check “Allow executing file as a program”.
  3. Double-click the .AppImage to run it.

Cursor App uses the same settings as VS Code. If you already use VS Code, switching is seamless — your existing config imports automatically on first launch.

4. Models & Usage

Cursor acts as an orchestrator — you write prompts in natural language, and Cursor routes them to an AI model that generates the response, code edit, or terminal command. You choose the model; Cursor handles the rest.

Each model differs in capability, speed, and token cost. Tokens are the units models use to measure input and output — longer prompts and larger files consume more tokens, and more capable models cost more tokens per request.

  • Recommendation: Just select Auto — it handles most tasks well by picking the right model for you.
  • Usage tracking: Monitor your token consumption under Settings > Usage to stay within your team's allocation.
  • Need more control? If Auto struggles with a complex task, try selecting a specific model from the comparison table below.

Model variations: Most models come in multiple variants you can select from the model picker:

  • Thinking mode — the model reasons step-by-step before answering (e.g., Claude Sonnet 4.6 with thinking). Higher quality but slower.
  • Fast mode — optimized for speed at higher cost (e.g., Claude Opus 4.6 Fast).
  • Max Mode — extends the context window up to 1M tokens for working with large codebases.
  • Reasoning effort — some OpenAI models let you choose low / high reasoning effort (e.g., gpt-5.3-codex-high).
  • Size variants — smaller/cheaper versions for simpler tasks (e.g., GPT-5.4 Mini, GPT-5.4 Nano).

Models and versions update frequently. Check the model picker in Cursor for the latest options, or visit cursor.com/docs/models-and-pricing.

5.2 Real-world examples

Rule of thumb: The more specific your prompt, the better the result. Tell Cursor exactly what you want — specify input files with @ references, name the output format, describe the plot style, and state the analysis steps. A vague prompt like “analyze my data” will give generic results; a precise prompt like “load @results.csv, run a Kruskal–Wallis test on column X grouped by column Y, and save a boxplot as figure.png” gets you there in one shot.

6. Saving Your Work with Git & GitHub

Git is a version control system that tracks every change you make — so you can undo mistakes and collaborate with others. GitHub is the online platform where your project is stored and shared. The good news: you don't need to learn Git commands — just ask Cursor in plain English.

Getting connected

  1. If you don't have one yet, create a free account at github.com.
  2. Open Cursor and ask it to help you set up GitHub:
text

The agent will install gh, run gh auth login, and walk you through the browser-based login. Once done, you can push code and create pull requests directly from Cursor.

St. Jude Git & GitHub resources

  • St. Jude GitHub – github.com/stjude. To request access to the St. Jude GitHub organization, submit a request through MyAccess (search for “GitHub”). See the ServiceNow KB article for step-by-step instructions.
  • St. Jude Git (internal) – git.stjude.org. Access can be requested by submitting a Research System General Request in ServiceNow.
  • For more details, see the RIS GitHub Guide on the St. Jude wiki.

What Cursor can do for you

  • Save a snapshot of your work (commit) with an auto-generated description
  • Upload your changes to GitHub (push)
  • Create a pull request so teammates can review your work
  • Show what changed since your last save
  • Undo recent changes or restore an earlier version

Just describe what you want in the chat or CLI:

text text

You can type @git in your prompt to give Cursor extra context about your recent changes, branches, or commit history — this helps it give more accurate answers.

Symptoms: Blank screen or crash on launch.

Solutions:

🚫 "agent: command not found" after CLI install

Cause: Shell PATH not updated.

bash
⏳ AI responses are slow or timing out
  • Check the model dropdown — larger models (e.g., Claude Opus 4.6) are slower but more capable.
  • Reduce context: don't @-reference huge directories; be specific.
  • Check status.cursor.com for service outages.
🔒 Privacy / data concerns

Privacy Mode is enabled by default. With Privacy Mode on:

  • Your code is not stored by any model provider.
  • Your code is not used for training.
  • Prompts are sent to the model for inference only and discarded.

You can verify this under Settings > Team Settings > Privacy.
↩️ Accepted a bad AI edit — how do I undo?
  • Press Ctrl+Z immediately to undo the last change, just like in any editor.
  • If the agent made changes across multiple files, use Checkpoint Restore — click the clock icon at the top of the chat panel to revert all files to the state before the agent ran.
  • If you already saved and closed, use git diff and git checkout -- filename to restore individual files (works only if your project is in a Git repo).
🤔 AI gives wrong or irrelevant answers

Cause: The model doesn't have enough context about your code.

Solutions:

  • Use @ references to pin specific files or folders (e.g., @analyze.py, @src/pipeline/) so the model reads them.
  • Be specific in your prompt — instead of "fix this", say "fix the KeyError in the process_data function in @utils.py".
  • Start a new chat if the conversation has gotten long — older context can confuse the model.
